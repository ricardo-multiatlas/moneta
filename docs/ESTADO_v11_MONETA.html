Cambios técnicos entregados en v0.11

Punto 2 · Exportación contable

  • Archivo nuevo: src/lib/exportar-contable.ts — 3 generadores CSV: A3 (importación libre ISO-8859-1), Contasol (CSV ISO-8859-1, decimal coma), Sage 50 (UTF-8 con BOM).
  • Modificado: src/routes/facturacion.tsx — dropdown con los 3 formatos + Excel genérico legacy.
  • Tratamiento fiscal: comisiones de mediación tratadas como EXENTAS de IVA (art. 20.Uno.16 LIVA, clave E2). Cada factura genera asiento de doble partida (Debe cliente + Haber ingresos) o triple si IVA > 0.
  • Cuentas PGC sugeridas: 43000001 (clientes), 70500000 (ingresos comisiones), 47700000 (IVA repercutido cuando aplica), 57200000 (bancos).

Punto 3 · Conciliación bancaria semi-automática (Norma 43 AEB)

  • Archivos nuevos: src/lib/n43-parser.ts, src/routes/conciliacion-bancaria.tsx.
  • Migración SQL nueva: supabase/migrations/20260621220000_v11_conciliacion_bancaria.sql con tablas movimientos_n43 y movimientos_n43_match + RLS por rol.
  • Parser: registros 11/22/23/33/88, ISO-8859-1, longitud fija 80 chars, fechas AAMMDD.
  • Scoring 0-100: importe exacto +40 (bloqueante), fecha valor ±3 días +20, NIF en concepto +25, número recibo/póliza +25, nombre tomador +10, código común coherente +5.
  • Umbrales: ≥85 y candidato único → auto-confirma + recibo pasa a "cobrado". 65-84 o empate → dudoso (revisión humana). <65 → ignorado.
  • Sidebar: nueva entrada "Conciliación bancaria" visible para root, admin y jefe_zona.

Punto 4a · Migración IA · Gemini → Mistral La Plateforme

  • Modificados: src/lib/ai-search.ts, src/lib/ai-comisiones.ts, src/routes/polizas.tsx (sólo extractPolicyFn).
  • Modelos: mistral-medium-latest (extracción PDF de pólizas y de informes de comisiones), mistral-small-latest (clasificación + filtros de búsqueda natural).
  • SDK: @ai-sdk/mistral oficial Vercel (drop-in del anterior @ai-sdk/google).
  • Variables entorno: nueva MISTRAL_API_KEY; eliminada GOOGLE_GENERATIVE_AI_API_KEY.

Punto 4b · Migración email · Resend → Brevo

  • Modificados: supabase/functions/enviar-aviso-vencimiento, supabase/functions/procesar-campana, supabase/functions/webhook-resend (carpeta conservada para no romper redirects).
  • Migración SQL nueva: supabase/migrations/20260621221000_v11_email_provider_brevo.sql renombra email_eventos.resend_id → provider_msg_id y añade columna provider con default 'brevo'.
  • API: https://api.brevo.com/v3/smtp/email con header api-key. Webhooks idénticos (delivered, opened, click, hard_bounce, soft_bounce, spam, unsubscribed) mapeados a tipos internos (entregado, abierto, click, rebote, spam, baja).
  • Variables entorno: nuevas BREVO_API_KEY, BREVO_FROM_EMAIL, BREVO_FROM_NAME; eliminadas RESEND_API_KEY, RESEND_FROM_EMAIL.
  • Frontend: referencias a "Resend" sustituidas por "Brevo" en comunicaciones.tsx, configuracion.tsx, configuracion.webhooks.tsx, presupuestos.tsx.
